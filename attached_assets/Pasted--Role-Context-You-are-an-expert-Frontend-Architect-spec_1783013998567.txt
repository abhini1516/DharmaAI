# Role & Context
You are an expert Frontend Architect specializing in React, TypeScript, Vite, and Tailwind CSS v4. Your task is to build a production-ready, highly polished frontend application named **Dharma.ai** — a Legal Literacy & AI-Driven Lawyer Matchmaking Platform tailored for the Republic of India (National & Central Laws). 

The platform must look like a high-end, production-grade web application with real data flows handled via a mock service layer in TypeScript, allowing for immediate backend integration (FastAPI + MongoDB + Gemini API + Vector RAG) by simply swapping out the async functions.

# Detailed Requirements & Interactive State Logic

## 1. Mock Data Layer (`src/lib/mock/`)
Provide real, structurally sound Indian statutory and constitutional data:
- **Topics**: Handle at least 5 deep categories: Constitution of India (Art. 21, 14), Bharatiya Nyaya Sanhita (BNS 2023 - replacing IPC, e.g., §106 negligence), Digital Personal Data Protection (DPDP) Act 2023, Consumer Protection Act 2019, and the IT Act 2000. Give each a `slug`, `summary`, `plainLanguageExplainer`, structured `keySections`, and `faqs`.
- **Lawyers**: Array of ~20 realistic profiles including: Name, Bar Council Registration Number, State/City (e.g., New Delhi, Bengaluru, Mumbai), Languages Spoken, Years of Experience, Practice Areas, Hourly Fees (INR), Rating, and a list of notable case wins.
- **AI Service (`services/ai.ts`)**: Implement an async function `askLegalQuestion(query: string)` that yields a typewriter-style delayed execution resolving to an answer string containing real citation tags (e.g., `[BNS-106]`) and 3 dynamic `suggestedFollowUps`.
- **Match Service (`services/match.ts`)**: Implement an async function `matchLawyers(caseText: string, filters: any)`. Simulate vector similarity math by returning a ranked subset of lawyers with a calculated `matchScore` (e.g., 94%) and an explicit, AI-generated text block for `matchReasoning` showing *why* their background matches the query.

## 2. Page & Flow Requirements

### Page 1: Home (`/`)
- Hero section with a prominent, centralized search omnibar ("Search bills, rights, or describe your issue...").
- Grid of featured Citizen's Library cards with subtle hover scaling.
- A split section previewing the AI Chat Assistant and a high-match sample lawyer card.

### Page 2: Library Index & Topic Reader (`/library` & `/library/:topicSlug`)
- **Index**: A clear, tabular grid mapping all 10 legal domains using icon emblems.
- **Reader**: Split editorial view. Left side: The Act overview written in completely plain, citizen-friendly language alongside key statutory articles. Right side: Interactive FAQ accordion, matched local lawyers for this domain, and an inline CTA box: "Have a specific question about this? Ask Dharma AI." Clicking it routes to the Consult page with the topic pre-loaded.

### Page 3: Consult / AI Chat (`/consult`)
- Clean, vertical message layout with no distracting clutter.
- Messages stream text in using a `useEffect` interval to simulate live LLM generation.
- Bot replies must convert text citations like `[BNS-106]` dynamically into custom `<CitationChip />` elements. Clicking a chip opens an inline side sheet displaying the exact verbiage of that specific section.
- Display clickable follow-up prompt chips below the newest response.

### Page 4: Lawyer Directory & Profiles (`/directory` & `/directory/:lawyerId`)
- **Directory**: Left-side sticky sidebar control suite supporting complex filters: Practice Area dropdown, State/City checkboxes, Languages, Experience slider, and Fee Sort. Right-side updates instantly on state changes showing sleek cards.
- **Profile**: A full editorial resume view showcasing verification badges, full biographical timeline, historical case outcomes, active bar memberships, and client reviews.

### Page 5: AI Matchmaking Wizard (`/match`)
A clean multi-step state machine (`currentStep` 1 to 3):
- **Step 1: Incident Description**: A deep textarea input inviting the user to describe their case in natural language ("Tell us what happened in plain terms...").
- **Step 2: Legal Preferences**: Prompt for clarifying parameters (Location, Language, Budget Range, Target Timeline).
- **Step 3: Evaluation Engine**: Upon clicking "Generate Matches", show a loading state with a simulated processing ticker ("Analyzing statutory crossover...", "Evaluating vector proximity to specialized bar practitioners..."). Once resolved, display the ranked layout with an explicitly highlighted "Match Reasoning Score" badge and custom justification sentences.
